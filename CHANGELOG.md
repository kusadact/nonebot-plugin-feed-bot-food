# CHANGELOG

本文件只记录插件本身的功能更新与 bugfix。

## [Unreleased]

### Changed

- 将每日结算改为基础代谢阈值和非线性摄入转换模型，并新增 `FEED_BOT_FOOD__METABOLIC_CONSTANT` 配置；移除旧的 `DECAY_FLUCTUATION` 配置。
- 将 `FEED_BOT_FOOD__CATEGORY_LIMITS` 改为每个窗口的总成功投喂次数，不再按食物类别分别限额。
- 将 LLM 请求硬上限改为总投喂次数的 `1.5` 倍向上取整。
- 移除窗口边界保护区和分类槽位顺延规则。
- 状态查询增加昨日成功投喂总次数和昨日净体重变化。
- 将状态文件格式升级为 schema v2，并在当前版本首次读取时自动迁移 schema v1；后续版本将移除旧格式兼容。

## [0.1.0] - 2026-07-13

### Added

- 增加 OneBot 群聊投喂和体重状态查询。
- 增加基于 OpenAI 兼容接口的食物分类与增重判断。
- 增加固定窗口分类额度、LLM 请求硬上限和窗口边界槽位顺延。
- 增加 JSON 持久化、每日 06:00 减重和离线结算补偿。
- 增加可选的 groupmate-agent `feed_bot_food` 与 `get_feed_bot_status` Tools。

### Fixed

- 增加超量食物识别：超过类别上限时只按最大限制投喂，并由 Agent 根据实际摄入量自然回复。
- 增加 `GAIN_RANGE_FLUCTUATION` 配置，对每次分类的增重范围上下限分别施加随机浮动，默认 `0.15kg`。
- 调整结果提示：无法确认分类时返回未投喂提示，LLM 请求或分类失败时也返回用户可见的失败提示。
- 明确 Agent Tool 对所有已调用的投喂结果都必须要求 Agent 回复用户，不再对分类或 LLM 请求失败静默处理。
- 增加投喂服务和直接命令的异常兜底，避免投喂处理异常导致用户无回复。
- 明确 LLM 对多个食物输入只按主要类别计算一次增重，不拆分为多个投喂。
- 防止 LLM HTTP 错误响应正文泄漏 API Key。
- 将 LLM 3xx 响应按请求失败处理。
- 修正多槽位额度的下一次可用时间计算。
- 允许空投喂命令返回参数提示。
- 将直接投喂和状态查询改为标准 `/投喂`、`/查看体重` 命令，并增加 `/查看状态` 别名；不再要求艾特 Bot。
